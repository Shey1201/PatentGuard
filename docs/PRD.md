# PatentGuard 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品定义

| 项目 | 内容 |
|------|------|
| 产品名称 | PatentGuard |
| 产品类型 | Web 应用（SaaS） |
| 一句话定义 | 基于知识库与 LLM 的企业级文档合规审查系统 |
| 目标用户 | 企业法务、专利工程师、合规部门、律所 |

### 1.2 解决问题

| 痛点 | 解决方案 |
|------|----------|
| 人工审查效率低 | 自动化 AI 审查，快速出结果 |
| 审查标准不统一 | 知识库统一规则，输出标准化 |
| 缺乏可追溯依据 | 引用来源可查，结果可解释 |
| API 配置不灵活 | 用户可自定义 LLM API |

### 2.3 文档审查模块

#### 2.3.1 审查功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 上传待审文档 | P0 | 支持 PDF、DOCX、TXT |
| 自动审查 | P0 | RAG + LLM 智能分析 |
| 审查结果展示 | P0 | 合规/风险、风险点、法规依据 |
| 审查历史 | P0 | 查看历史审查记录 |
| 结果导出 | P1 | PDF/Word 导出报告 |

#### 2.3.2 审查输出

```json
{
  "compliance": true,
  "risk_level": "low",
  "summary": "该文档符合相关规定",
  "findings": [
    {
      "type": "risk",
      "severity": "low",
      "description": "缺少必要的技术交底书",
      "reference": "专利法实施细则第XX条",
      "suggestion": "建议补充技术交底书"
    }
  ],
  "referenced_documents": [
    {
      "title": "专利法",
      "relevance": 0.95,
      "matched_chunks": ["第一条 ...", "第二条 ..."]
    }
  ]
}
```

## 三、非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| API 响应时间 | < 2s（不含 LLM 调用） |
| 文档上传大小 | 最大 50MB |
| 向量检索延迟 | < 500ms |
| 支持并发用户 | 50+ |

### 3.2 安全需求

| 项目 | 要求 |
|------|------|
| 密码存储 | 哈希存储（bcrypt） |
| 认证方式 | JWT Token |
| Token 有效期 | 24 小时 |
| API 限流 | 100 次/分钟 |
| 数据传输 | 生产环境需 HTTPS |

### 3.3 可用性需求

| 项目 | 要求 |
|------|------|
| 系统可用性 | 99.9% |
| 错误恢复 | 异常日志记录、可重试 |
| 数据备份 | 每日自动备份 |
