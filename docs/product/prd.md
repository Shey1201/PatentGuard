# PatentGuard PRD

## 产品定位

PatentGuard 是一个基于知识库和 RAG 的文档合规审查系统，面向专利、合同、法规和通用文档场景，帮助用户完成资料入库、检索增强审查、风险提示和引用溯源。

## 目标用户

- 法务、知识产权、合规审查人员
- 需要批量审查专利/合同/政策文档的团队
- 需要自带模型服务和知识库的企业用户

## 核心问题

| 问题 | 解决方式 |
| --- | --- |
| 文档资料分散 | 建立可检索的知识库 |
| 人工审查效率低 | 使用 RAG + LLM 辅助生成审查结果 |
| LLM 输出缺少依据 | 输出引用来源，并进行基础引用校验 |
| 模型服务配置不灵活 | 支持用户自定义 LLM 与 Embedding 配置 |

## 核心功能

### 知识库管理

- 文档上传
- 文本解析与分块
- Embedding 向量化
- 分类管理
- 语义检索

### 文档审查

- 上传文件并发起审查
- 选择审查类型
- 轮询审查状态
- 查看风险等级、摘要、问题列表和引用来源
- 查看历史审查记录

### 自定义模型配置

系统不绑定固定 LLM 厂商。用户需要配置：

| 配置项 | 说明 |
| --- | --- |
| API Key | 模型服务密钥 |
| Base URL | 兼容 Chat Completions 调用格式的接口基础地址 |
| Model | 模型服务实际支持的模型名称 |

后端按以下格式调用：

```text
POST {base_url}/chat/completions
```

Embedding 也使用独立配置，包括模型名、接口地址、API Key 和向量维度。

### 埋点与统计

- 页面访问
- 上传、审查、搜索、配置等关键业务事件
- API 调用日志
- 管理员统计看板

## 非功能需求

| 类型 | 要求 |
| --- | --- |
| 可配置性 | LLM、Embedding、数据库、存储均通过环境变量或用户配置注入 |
| 可追溯性 | 审查结果保留引用来源 |
| 可观测性 | 记录 API 调用日志和业务事件 |
| 可维护性 | 前后端分层，文档按读者分类 |

## 当前边界

- LLM 接入依赖目标服务兼容 Chat Completions 格式。
- 不维护固定厂商白名单。
- 前端测试体系尚未完善。
- Docker Compose 当前主要面向本地开发。
