## 三、数据库设计

### 3.1 表结构

#### BusinessEvent 表 - 业务事件表

```sql
CREATE TABLE business_events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    event_name VARCHAR(100) NOT NULL,      -- 事件名称
    event_category VARCHAR(50),            -- 事件类别
    user_id UUID,                         -- 用户 ID
    session_id VARCHAR(100),               -- 会话 ID
    resource_id UUID,                      -- 关联资源 ID
    properties JSONB,                      -- 自定义属性
    system_info JSONB,                     -- 系统信息
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_business_events_event_name ON business_events(event_name);
CREATE INDEX idx_business_events_user_id ON business_events(user_id);
CREATE INDEX idx_business_events_session_id ON business_events(session_id);
CREATE INDEX idx_business_events_created_at ON business_events(created_at);
```

#### APILog 表 - API 调用日志表

```sql
CREATE TABLE api_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID,                         -- 用户 ID
    method VARCHAR(10),                   -- HTTP 方法
    endpoint VARCHAR(255),                -- API 端点
    status_code INTEGER,                  -- 响应状态码
    duration_ms INTEGER,                  -- 响应时间（毫秒）
    request_size BIGINT,                  -- 请求大小
    response_size BIGINT,                 -- 响应大小
    ip_address VARCHAR(45),              -- 客户端 IP
    user_agent TEXT,                      -- 用户代理
    error_message TEXT,                   -- 错误信息
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_api_logs_endpoint ON api_logs(endpoint);
CREATE INDEX idx_api_logs_user_id ON api_logs(user_id);
CREATE INDEX idx_api_logs_status_code ON api_logs(status_code);
CREATE INDEX idx_api_logs_created_at ON api_logs(created_at);
```

### 5.2 Tracker SDK

**位置**: `frontend/src/utils/tracker.ts`

#### 主要功能

1. **会话管理**: 自动生成和管理会话 ID
2. **系统信息采集**: 设备、浏览器、操作系统、屏幕尺寸等
3. **事件队列**: 批量发送事件，减少网络请求
4. **自动发送**: 支持定时发送和页面卸载时发送
5. **用户关联**: 支持设置用户 ID 进行用户级别分析

#### 核心 API

```typescript
// 基础埋点
tracker.track(event: TrackEvent)

// 页面浏览
tracker.trackPageView(data: PageViewData)

// 点击事件
tracker.trackClick(data: ClickData)

// 文档上传
tracker.trackDocumentUpload(data: ReviewData)

// 审查提交
tracker.trackReviewSubmit(data: ReviewData)

// 审查完成
tracker.trackReviewComplete(data: ReviewData & { success: boolean; duration_ms: number })

// 搜索行为
tracker.trackSearch(query: string, resultCount: number)

// 登录
tracker.trackLogin(method: string, success: boolean)

// 登出
tracker.trackLogout()

// 错误
tracker.trackError(errorType: string, errorMessage: string, context?: Record<string, any>)

// 自定义事件
tracker.trackCustom(eventName: string, properties?: Record<string, any>)
```

#### 使用示例

```typescript
import { tracker } from '@/utils/tracker';

// 记录页面浏览
tracker.trackPageView({
  page_url: '/review',
  page_title: '文档审查',
  referrer: document.referrer,
});

// 记录按钮点击
tracker.trackClick({
  element_id: 'btn-submit',
  element_text: '开始审查',
  page_url: window.location.pathname,
});

// 记录文档上传
tracker.trackDocumentUpload({
  document_name: '专利申请书.pdf',
  review_type: 'patent',
  file_size: 1024000,
  file_type: 'pdf',
});
```

### 6.4 埋点事件设计原则

为了确保埋点数据有意义且可转化，我们遵循以下设计原则：

#### 6.4.1 有意义的埋点标准

1. **可度量**：每个事件都有明确的业务指标对应
2. **可归因**：事件可以追溯到具体的业务目标
3. **可优化**：基于数据可以采取优化行动
4. **不过度**：避免无意义的重复埋点

#### 6.4.2 事件命名规范

- 页面浏览：`{page}_{action}`，如 `kb_document_view`
- 业务操作：`{module}_{action}`，如 `review_submit`
- 系统事件：`{type}_{name}`，如 `performance_sample`

#### 6.4.3 属性设计原则

- **必填属性**：用于核心分析的字段
- **可选属性**：用于深度分析的字段
- **避免过度采集**：只采集必要的数据

### 6.5 系统信息字段

| 字段 | 说明 | 示例 |
|------|------|------|
| device | 设备类型 | Desktop, Mobile, Tablet |
| browser | 浏览器 | Chrome, Firefox, Safari, Edge |
| os | 操作系统 | Windows, macOS, Linux, iOS, Android |
| screen_size | 屏幕分辨率 | 1920x1080 |
| language | 浏览器语言 | zh-CN, en-US |
| timezone | 时区 | Asia/Shanghai, America/New_York |

## 八、数据安全与隐私

### 8.1 隐私保护措施

1. **数据脱敏**: 敏感信息（如邮箱）在存储前进行脱敏处理
2. **HTTPS 传输**: 所有埋点数据通过 HTTPS 加密传输
3. **访问控制**: API 日志仅管理员可查询
4. **数据保留**: 建议设置合理的数据保留期限（如 90 天）

### 8.2 合规建议

1. 在用户协议中明确告知数据收集范围
2. 提供用户数据删除机制
3. 定期审计数据访问权限
4. 加密存储敏感数据

## 九、性能优化

### 9.1 前端优化

1. **批量发送**: 事件先入队列，达到阈值或定时触发后批量发送
2. **页面卸载处理**: 使用 `navigator.sendBeacon` 确保页面卸载时数据不丢失
3. **异步发送**: 不阻塞主线程，不影响用户体验
